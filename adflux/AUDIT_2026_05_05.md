## SECTION 2: MODULE GAPS

### 1. Authentication + Role Management
**What exists:**
- OAuth via Supabase Auth (`auth.users` table, sign-in/sign-out)
- Role column on `users` table (admin, sales, co_owner, agency)
- Segment access: `segment_access` on users (PRIVATE / GOVERNMENT / ALL)
- Signing authority flag: `signing_authority` on users (drives govt signer picker)
- Role-based UI gating via `RequireAdmin`, `RequirePrivileged`, `RequireGovtAccess`
- RLS policies on all tables (admin-write, sales-own patterns)

**Missing:**
- Multi-tenant role hierarchy (e.g., team lead / team member) — all sales are flat
- Permission templates or fine-grained ACLs beyond role + segment
- Audit log of who changed what when (no `created_by` / `updated_by` / `updated_at` on most tables)
- User deactivation workflow (soft-delete only via `is_active` boolean)

### 2. Quotes (Private LED)
**What exists:**
- Full quote creation wizard (5 steps: client, campaign, cities, review, send)
- Quote status flow: draft → sent → negotiating → won / lost
- Quote line items (`quote_cities`) with rate overrides
- Quote PDF generation (via `QuotePDF.jsx`)
- Segment scoping: private quotes all have `segment='PRIVATE'` or `NULL` (legacy)

**Missing:**
- Quote versioning / revision tracking (no `version` or `revision_number` column)
- Quote approval workflow before sending (no intermediate "pending approval" state)
- SKU / product templates for cities (cities are pre-defined only; no bundling)

### 3. Quotes (Government Auto Hood)
**What exists:**
- 5-step wizard (recipient, date/signer, quantity, districts, review)
- DAVP rate lookup (hardcoded national rate)
- District selection with quantity distribution
- PDF proposal renderer (HTML → printable letter format)
- Segment: `segment='GOVERNMENT'`, `media_type='AUTO_HOOD'`, `rate_type='DAVP'`

**Missing:**
- Creative/design attachment workflow (no design PDF upload or versioning)
- Tracking of which districts were targeted vs. actual execution
- Auto-renewal / repeat order templates for same govt client

### 4. Quotes (Government GSRTC LED)
**What exists:**
- 5-step wizard (recipient, date/signer, stations, months, review)
- Station selection per city
- Campaign month duration
- PDF proposal renderer
- Segment: `segment='GOVERNMENT'`, `media_type='GSRTC_LED'`

**Missing:**
- Station inventory / capacity management (no "slots booked vs. available")
- Media rotation schedule (no timeline of which creative runs when)
- Conflict detection (overlapping campaigns at same station)

### 5. Clients
**What exists:**
- `clients` table with fields: name, company, phone, email, GST, address, notes
- Client list view at `/clients` (RLS scopes per sales rep)
- Quick "new quote from client" action
- Admin sees all; sales sees own

**Missing:**
- Client relationship tracking (Assigned rep / Account manager rotation)
- Contact person CRUD (multi-contact per client — all fields in one `clients` row now)
- Client segmentation (priority / category / industry)
- Complaint/ticket workflow linked to client

### 6. Payments + Approvals
**What exists:**
- `payments` table with: amount, mode (NEFT/RTGS/UPI/Cheque/Cash), date, reference, notes, received_by, is_final_payment, approval_status (pending/approved/rejected)
- Payment approval workflow: sales submits → admin approves/rejects
- On final payment approval: quote status → won, incentive credited
- Realtime channel subscription on `/pending-approvals` (live updates)
- Multiple payments per quote allowed

**Missing:**
- Payment plan / instalment templates (no "split 50/30/20" presets)
- Payment receipt generation (no PDF receipt)
- Refund / credit memo workflow (no refund entity)
- Email notification on approval (mailto: links only, no SMTP send)

### 7. Follow-ups
**What exists:**
- `follow_ups` table: quote_id, follow_up_date, follow_up_type, notes, done, created_by
- Follow-up list page (v1 stub, not in v2 yet)
- Follow-up banner on dashboard shows due-today count
- RLS: sales see own, admin sees all

**Missing:**
- Follow-up scheduler / bulk assignment
- Follow-up activity log (no "called at 3pm, will call back tomorrow" tracking)
- Follow-up templates (no "check in on proposal", "post-renewal outreach")
- Integration with CRM calendar (no iCal or Google Calendar sync)

### 8. Master Config
**Exists:**
- Cities (private LED): `/cities` CRUD, rate + override management (Phase 9b seeded private LED cities)
- Auto Districts (govt): `/auto-districts` CRUD, read-only DAVP rate
- GSRTC Stations (govt): `/gsrtc-stations` CRUD, rate per station per city
- Companies (Phase 10): segment → (GOVERNMENT / PRIVATE) with GSTIN, bank, address, logo, letterhead; seeded from real invoices
- Signers (Phase 5): users with `signing_authority=true`, can have signature_mobile + signature_title override
- Document templates (Phase 8D): proposal_templates table with HTML body + versioning
- Attachments/Media (Phase 8C, Phase 11E): Master page structure stubbed but incomplete

**Missing:**
- Logo/brand asset management (companies.logo_url exists but no upload UI in Master)
- Email signature templates
- Invoice number sequencing (no auto-increment per segment)
- Tax bracket / GST rate versioning (hardcoded 18% in code)

### 9. Team Management
**What exists:**
- Team member list: name, email, role, segment_access, signing_authority, is_active
- CRUD modal on `/team` page
- Role assignment (admin / sales / co_owner / agency)
- Segment access assignment (PRIVATE / GOVERNMENT / ALL)
- Signing authority toggle (for govt signers)

**Missing:**
- Team hierarchy / reporting structure (no manager_id column)
- Location assignment (no user_location or territory)
- Skills / specialization tags
- Performance data rollup (team lead view of direct reports)

### 10. HR (Offers, Onboarding)
**What exists:**
- `hr_offers` table: candidate_name, candidate_email, position, territory, offer_token, status (draft/sent/filled/accepted/converted_to_user/cancelled)
- Send offer letter (templated HTML PDF)
- Candidate fills form via public `/offer/:token` link
- On form submit: offer_status → filled
- Admin can convert filled offer → new sales user

**Missing:**
- Offer template versioning (one template per position only)
- Document checklist for onboarding (KYC, bank, insurance)
- Salary slip templates / payroll integration
- Exit workflow / resignation tracking

### 11. Renewals
**What exists:**
- Renewal Tools page: lists won quotes with campaign_end_date within 60 days
- 3-bucket color code: <7 days (red), <30 days (amber), else green
- Admin sees all; sales sees own
- UI: shows client, quote number, amount, days remaining

**Missing:**
- Automatic renewal quote generation (no "clone this quote" feature)
- Renewal discount / terms templates
- Renewal override (e.g., "customer wants new start date")
- Bulk renewal action (select multiple, generate quotes in batch)

### 12. Incentives
**What exists:**
- `staff_incentive_profiles` table: user_id, slab_1_2_3_4_5 (revenue targets), payout_1_2_3_4_5 (rupee amounts)
- `incentive_settings` table: (admin config of slab structure)
- Monthly revenue rolling calculation (`useIncentive.js` hook)
- Admin dashboard: team slabs, monthly rollups, liability forecast, What-If simulator
- Sales view: My Performance → monthly target progress, slab position
- Payout modal: mark user as paid, track payment_date + amount

**Missing:**
- Incentive dispute / adjustment workflow (no approval for manual tweaks)
- Commission clawback (no mechanism to recalculate if quote lost after payout)
- Incentive carry-over (no "shortfall from prior month" buffer)
- Segment-specific incentive rates (both segments use same slab now)
- Advanced features: tiered team bonuses, top performer bonuses, sales manager commissions

### 13. Dashboards
**Admin Dashboard (working):**
- Revenue summary (YTD, MTD, month-on-month)
- Pipeline funnel (draft → sent → negotiating → won)
- Active campaigns (count, revenue, client breakdown)
- Top performers (rank by monthly revenue)
- Pending approvals banner
- Rejection banner (lost quotes summary)
- Renewal reminder banner
- Follow-ups due today
- Incentive liability forecast
- Activity feed (realtime quote/payment updates)
- Outstanding payments (unpaid won quotes)

**Sales Dashboard (working):**
- Personal revenue summary (MTD, target, gap)
- My active quotes (won + in-progress)
- Follow-ups due today
- Renewal reminders (my campaigns ending)
- Next action alerts (pending approvals, follow-up due)
- Activity feed (my quote updates)

**Missing Dashboard Features:**
- Segment split (govt vs. private revenue share)
- Territory/area breakdown (no location field on sales)
- Customer acquisition cost analysis
- Win/loss analysis (why did we lose these quotes?)
- Rep-to-target comparison (all sales vs. their target)
- Forecast / pipeline health (what's likely to close this month?)

## SECTION 4: CROSS-ROLE GAPS

### Pages With Admin-Only UI, No Sales Variant
| Feature | Admin Page | Sales Variant | Issue |
|---------|-----------|---|---|
| Incentive Slab Configuration | `/incentives` (IncentiveSettings modal) | None | Sales can't see how slabs work; only see personal progress |

### UI Exposed, RLS May Block
| Feature | UI | RLS Check | Risk |
|---------|----|----|------|
| Quote detail payment tab | All roles see it | Only admin can approve | Sales users see "pending" but can't understand approval flow |
| Client list search | All roles see it | RLS filters correctly | OK — UI matches data access |
| Govt proposal detail view | Sales sees proposal renderer | No explicit admin check; RLS on quote | Risk: Phase 11J log shows fetch errors on hard reload due to RLS race |

### RLS Allows, UI Doesn't Expose
| Capability | RLS Policy | Frontend | Impact |
|-----------|-----------|----------|--------|
| Update own quote after sent | Yes, sales can UPDATE | CreateQuote* assumes draft-only | Can't edit quote name/client after send (minor) |
| Bulk quote status change | Yes, admin can UPDATE | No bulk action UI | Admin must click each quote individually |
| Download all quotes as CSV | Yes, data is readable | No CSV export | No batch reporting (missing) |
| Query team performance by date range | Yes, via `created_at` | No date-range filter in dashboard | Static MTD only |
