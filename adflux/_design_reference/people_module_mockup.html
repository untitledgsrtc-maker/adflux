  • /team, /incentives, /salary, /leaves still work (redirect to /people?tab=…)
  • No new tables, no new RLS
  • One sidebar entry instead of four
Proposal scope. Consolidate 4 admin pages into one People module with sub-tabs. Use the Incentives-page UI vocabulary (summary cards + filter row + panel table + modal) consistently. No functional change — every RPC, every modal, every column behaviour preserved.

What ships in Phase 38. 1 new page `PeopleV2.jsx` with 4 tabs that mount the existing page contents as sub-components. Sidebar entry consolidation. Deep-link redirects from /team /incentives /salary /leaves.

What does NOT ship. No DB change. No RPC change. No new master config. No incentive-formula change. No salary-formula change. No new payout flow beyond Phase 37 (already merged).
